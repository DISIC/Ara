# Workflow git

## Prise en main d’un ticket

- 1 branche = 1 PR = 1 ticket
- colonnes du kanban
- assignation sur le ticket
- création de la branche
- discussion technique en amont si besoin

## Commit

- nommage du commit
- 1 commit = 1 changement
- "atomic commit"

## Pull request

- nom de la pr
- description
- tag des reviewers

## Review et merge

- avant de mettre en review :
  - est-ce que tout est prêt : je réponds au scope du ticket.
  - est-ce que j'ai relu ma PR : les TODO, les `.only`, les typos...
  - est-ce que j'aimerai review ma propre PR
  -
- mettre à jour le changelog si besoin avant de merge

## Conflits

- rebase
- pour éviter :
  - rebase régulièrement
  - PR petit scope
